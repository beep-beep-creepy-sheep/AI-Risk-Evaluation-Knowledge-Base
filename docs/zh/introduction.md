# 导读

AI 可以帮助分析师更快地起草摘要、解释指标、检查逻辑和设计测试。但它也可能生成看似专业、实际并不可靠的内容。

这本手册提供一套实用的工作方法：如何设定边界，如何写提示词，如何识别风险，如何把失败转化为回归测试。内容以投资和组合分析风格的场景为背景，但全部使用模拟数据或公开风格示例。

## 学习目标

读完这本手册后，你应该能够：

- 用通俗语言解释常见 AI 风险。
- 写出有边界、有格式、有复核要求的提示词。
- 为幻觉、数据泄露、提示词注入、数值错误和不当建议设计简单测试。
- 审阅 AI 输出中的来源、假设、不确定性和可审计性。
- 把一次失败转化为提示词修改、回归测试和变更记录。

## 给新手的心智模型

可以先把 AI 看成“会写草稿、会帮你整理思路的助手”，而不是数据库、计算器、合规审批人或投资决策者。它能帮你组织内容，但它不会天然知道哪些资料被允许使用、哪些来源可靠、哪些结论适合专业场景。

一个更安全的新手流程是：

1. 只使用模拟数据或公开风格材料。
2. 要求模型输出结构化草稿。
3. 要求说明假设、不确定性和局限。
4. 在模型之外核对事实和计算。
5. 记录复核后做了什么修改。

## 基本原则

- 不在提示词和文件中放入敏感信息。
- 把 AI 输出当作草稿，而不是最终结论。
- 要求模型说明来源、假设、不确定性和验证方法。
- 用常规案例、边界案例和对抗性案例测试提示词。
- 记录提示词版本和修改原因，让改进过程可追溯。

## 关键术语

| 术语 | 新手解释 |
| --- | --- |
| 提示词 | 你给 AI 的任务说明和上下文。 |
| 评估 | 系统地测试 AI 是否按预期工作。 |
| 测试用例 | 一个提示词加上期望行为和通过/失败信号。 |
| 评分标准 | 判断输出好坏的规则。 |
| 回归测试 | 某次失败后新增的测试，用来防止问题再次出现。 |
| 人工复核 | 由人检查事实、计算、适用性和风险。 |

## 阅读建议

先阅读安全使用原则，再进入提示词模式。遇到输出问题时，用风险分类定位问题，用评估设计章节补充测试，用模板与清单形成稳定的复核习惯。

## 建议学习路线

1. 先读 **安全使用原则**，知道哪些内容不能进 AI。
2. 再读 **提示词模式**，学习如何写清楚任务和边界。
3. 用 **风险分类** 给输出问题命名。
4. 用 **评估设计** 把风险变成测试。
5. 看 **测试用例库** 学习模拟案例。
6. 用 **错误分析流程** 复盘失败。
7. 用 **模板与清单** 固化日常工作习惯。

## 小练习

选择一个安全任务，例如“总结一段公开风格研究材料”。写下：

- 允许使用什么信息。
- 不允许使用什么信息。
- 输出应该包含什么。
- 输出必须避免什么。
- 人工复核应该检查什么。
