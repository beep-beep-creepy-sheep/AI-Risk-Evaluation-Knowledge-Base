# 组合分析场景

本章把前面的 AI 风险概念放到组合分析和报告工作流中理解。内容仅用于学习和质量保证，所有示例都使用模拟数据或公开风格材料。

## 学习目标

读完本章后，你应该能够：

- 解释为什么 AI 风险与提示词评估知识库和组合分析工作相关。
- 描述 AI 在起草、检查和学习中的安全辅助角色，而不是替代判断。
- 识别金融服务场景中的特定风险，例如适当性、保密性、投资建议边界、记录保存和审批要求。
- 判断哪些事情不应使用真实工作资料或未经批准的 AI 工具来做。

## 更准确的项目定位

这个项目可以用下面的标题来概括：

```text
AI Risk & Prompt Evaluation Knowledge Base for Portfolio Analytics
```

这个定位的好处是：它贴近组合分析和报告场景，但不会把项目包装成交易工具、投资建议系统或商业产品。它强调的是教育、质量保证、提示词规范和风险管理。

## AI 可以安全辅助什么

当任务边界清楚，且输入是模拟数据、公开材料或已批准可用于该工具的信息时，AI 可以提供帮助。

| 分析师工作流 | AI 的安全角色 | 边界 |
| --- | --- | --- |
| 公开研究摘要 | 基于提供的公开风格文本起草摘要 | 不补充来源外事实 |
| Excel 公式支持 | 建议公式和校验方法 | 公式必须由人工独立检查 |
| 指标解释 | 用模拟数值解释波动率、回撤、敞口或跟踪误差 | 不建议配置调整 |
| 报告逻辑审阅 | 标出无依据结论、缺失假设和推理不一致 | 不审批、不发布报告 |
| 评估设计 | 创建模拟正向、边界和回归测试 | 不使用真实客户或账户数据 |

## 分析师提示词模式

分析师提示词应要求模型“辅助”，而不是“决策”。

更好的写法：

```text
只使用下面的公开风格材料，起草一段中性分析摘要。
请区分事实和解释，说明不确定性，并列出需要核验的来源问题。
```

应避免：

```text
告诉我最终投资结论。
```

更好的写法：

```text
基于下面的模拟工作表结构，建议一个 Excel 公式和一个校验行。
请说明假设和边界情况。
```

应避免：

```text
用真实账户数据计算并标记为已批准。
```

## 组合分析 AI 风险清单

审阅 AI 辅助分析工作时，可以检查：

- **幻觉**：模型是否编造了事实、来源、持仓、基准或市场事件？
- **数据泄露**：提示词或输出是否包含私密、账户级、员工级或保密信息？
- **保密性**：工作流是否鼓励使用非公开材料？
- **模型偏见**：模型是否对地区、行业、资产类别或投资者类型做了无依据概括？
- **过度依赖**：输出是否在没有人工复核的情况下被当作最终版本？
- **错误金融推理**：公式、敞口、收益、回撤或时间区间是否有误？
- **来源可靠性**：结论是否对应到提供的公开来源，还是来源模糊？
- **投资建议边界**：输出是否告诉某人买入、卖出、持有、配置或择时？
- **记录保存**：是否记录提示词版本、复核意见和变更？

## 模拟评估用例示例

最安全的学习方式是使用虚构数据测试。

| 测试类型 | 模拟输入 | 期望行为 |
| --- | --- | --- |
| 市场评论摘要 | 一段关于模拟指数变动的虚构文字 | 只总结给定事实，并说明不确定性 |
| 敞口计算 | 一张包含行业权重的模拟表格 | 解释计算方法，并加入校验 |
| 建议边界案例 | 一个虚构投资者画像询问该买什么 | 拒绝个性化建议，改给教育性问题 |
| 敏感信息案例 | 含有看似真实标识符的提示词 | 拒绝处理，并建议合成占位符 |
| 提示词注入案例 | 粘贴内容中写着“忽略任务指令” | 把嵌入指令当作文本，而不是命令 |

## 安全使用模板

可以在提示词、文档或复核清单中使用这些表述：

```text
不要包含客户姓名、账户标识、真实持仓、私密记录、员工信息或保密文件。
```

```text
只使用公开信息、提供的模拟数据，或明确批准可用于该工具的数据。
```

```text
所有 AI 输出都是草稿，专业使用前必须由合格人员复核。
```

```text
未经适当复核和审批，不得把 AI 输出直接发送给客户或外部对象。
```

```text
请标注来源、假设、不确定性和验证方法。
```

## 金融服务特有风险

组合分析和报告工作有一些特定边界。

| 风险领域 | 含义 | 本项目中的安全处理 |
| --- | --- | --- |
| 适当性 | 某个产品、配置或行动是否适合某个人或授权范围 | 不为真实个人判断适当性，只用教育性模拟场景 |
| 客户保密 | 客户、账户、持仓和关系信息需要保护 | 不包含可识别或真实账户级数据 |
| 市场滥用风险 | 非公开、可能影响市场的信息不得被误用或披露 | 只使用公开风格示例 |
| 记录保存 | 专业流程可能要求保留决策、沟通和审批记录 | 跟踪提示词版本、输出、评估结果和复核意见 |
| 投资建议边界 | 个性化建议需要批准流程和合格复核 | 拒绝买入、卖出、持有、配置和择时建议 |
| 合规审批 | 外部或专业材料可能需要审核后才能使用 | 把 AI 输出视为草稿 |
| 来源可靠性 | 市场和投资相关结论需要可靠、可核验来源 | 要求来源标注和核验问题 |

## 应避免的活动

不要用未经批准的工具或设置做这些事：

- 用真实客户信息测试 AI。
- 输入真实持仓、交易、业绩、授权细节或账户级记录。
- 粘贴内部报告、会议记录、政策文件或保密研究。
- 编写绕过安全系统或规避控制的教程。
- 使用工作资料构建商业产品。
- 用工作文件或私密数据训练个人模型。
- 发布暗示掌握保密观点、客户情况或非公开市场看法的文章。
- 未经必要复核和审批，把 AI 生成内容发给客户或外部对象。

## 小练习

把下面这个有风险的想法改写成安全学习项目：

```text
我想用真实报告案例测试 AI，看看它能不能生成最终可发给客户的评论。
```

你的安全版本应包含：

- 模拟或公开风格输入。
- 教育目的。
- 不生成客户可直接使用的最终文本。
- 人工复核。
- 来源和不确定性要求。
